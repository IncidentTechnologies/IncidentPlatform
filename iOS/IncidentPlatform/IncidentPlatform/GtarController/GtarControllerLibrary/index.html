The total number of LEDs on the Gtar, equal to the number of strings times the number of frets.

GtarControllerMaxLedIntensity

The maximum LED intensity. 

const NSInteger GtarControllerMaxLedIntensity = 3;
Discussion

Each color component of each LED has multiple discrete intensity levels. Each intensity varies from [0..GtarControllerMaxLedIntensity], where 0 is off and 'GtarControllerMaxLedIntensity' is full brightness.

GtarString

A guitar string. 

typedef unsigned char GtarString;
Discussion

A data type representing a string on the Gtar. The range of string values are [1..GtarControllerStringCount], corresponding to the strings of the Gtar device.

GtarFret

A guitar fret. 

typedef unsigned char GtarFret;
Discussion

A data type representing a fret on the Gtar. The range of fret values are [1..GtarControllerFretCount], corresponding to the frets of the Gtar device. A GtarFret value of 0 is a special value for the "open string", when a string is plucked without any fret being pressed. This is a common and valid value.

GtarLedIntensity

An LED intensity value. 

typedef unsigned char GtarLedIntensity;
Discussion

A data type representing the intensity of a single LED color on the Gtar. Acceptable values are in the range [0..GtarControllerMaxLedIntensity].

GtarPosition

A position on the fretboard. 

typedef struct { 
    GtarFret fret; 
    GtarString string; 
} GtarPosition;
Fields
fret

The GtarFret for this note.

string

The GtarString for this note.

Discussion

A GtarPosition is a string-fret pair, often used as a representation for a plucked note from the Gtar. The range of 'fret' and 'string' is [1..GtarControllerFretCount] and [1..GtarControllerStringCount] respectively. A 'fret' value of 0 is a special value for the 'open-string', when a string is plucked without any fret being pressed.

GtarLedColor

The RGB color components of a guitar LED. 

typedef struct { 
    GtarLedIntensity red; 
    GtarLedIntensity green; 
    GtarLedIntensity blue; 
} GtarLedColor;
Fields
red

The GtarLedIntensity for the red channel.

green

The GtarLedIntensity for the green channel.

blue

The GtarLedIntensity for the blue channel.

Discussion

A single Gtar LED is composed of three different color components. Different colors can be created by combining RGB components using standard color compositions for additive color. Each intensity varies from [0..GtarControllerMaxLedIntensity], where 0 is off and 'GtarControllerMaxLedIntensity' is full brightness. A GtarLedColor of (0,0,0) turns off an LED.

Common colors include:

Red = (GtarControllerMaxLedIntensity, 0, 0)

Green = (0, GtarControllerMaxLedIntensity, 0)

Blue = (0, 0, GtarControllerMaxLedIntensity)

White = (GtarControllerMaxLedIntensity, GtarControllerMaxLedIntensity, GtarControllerMaxLedIntensity)

Yellow = (GtarControllerMaxLedIntensity, GtarControllerMaxLedIntensity, 0)

Orange = (GtarControllerMaxLedIntensity, GtarControllerMaxLedIntensity/2, 0)

Purple = (GtarControllerMaxLedIntensity/2, 0, GtarControllerMaxLedIntensity/2)

Pink = (GtarControllerMaxLedIntensity, 0, GtarControllerMaxLedIntensity)

"Black" / Off = (0, 0, 0)

GtarLedColorMap

A string-to-GtarLedColor color mapping. 

typedef struct { 
    GtarLedColor stringColor[GtarControllerStringCount]; 
} GtarLedColorMap;
Fields
stringColor

Each string of the Gtar device has a GtarLedColor intensity associated with it.

Discussion

Define a per-string default color mapping. Each GtarLedColor in the 'stringColor' array maps to a string of the Gtar device. Note that C-style arrays are 0-based while GtarStrings are 1-based, so 'stringColor' index [0..GtarControllerStringCount-1] maps to GtarString [1..GtarControllerStringCount].

GtarControllerStatus

The result status code used by the GtarController. 

enum GtarControllerStatus { 
    GtarControllerStatusOk = 0, 
    GtarControllerStatusError, 
    GtarControllerStatusNotConnected, 
    GtarControllerStatusInvalidParameter 
};
Constants
GtarControllerStatusOk

No errors occurred.

GtarControllerStatusError

An error occurred. Running with a 'logLevel' of 'GtarControllerLogLevelError' or greater will output additional details about the error.

GtarControllerStatusNotConnected

An error occurred because the Gtar device was not connected. Try connecting to the device to resolve the error.

GtarControllerStatusInvalidParameter

An error occurred because of an invalid parameter. Running with a 'logLevel' of 'GtarControllerLogLevelError' or greater will output specific information about the invalid parameter.

Discussion

These are the values that are returned by the GtarController. A status of 'GtarControllerStatusOk' means success, any other value is an error.

GtarControllerLogLevel

The detail level of log output 

enum GtarControllerLogLevel { 
    GtarControllerLogLevelOff = 0, 
    GtarControllerLogLevelError, 
    GtarControllerLogLevelWarn, 
    GtarControllerLogLevelInfo, 
    GtarControllerLogLevelAll,  
};
Constants
GtarControllerLogLevelOff

Do not log any messages.

GtarControllerLogLevelError

Only log messages relating to errors (e.g. messages relating to 'GtarControllerStatusError').

GtarControllerLogLevelWarn

Log messages relating to non-critical errors.

GtarControllerLogLevelInfo

Log informational messages.

GtarControllerLogLevelAll

Log all messages.

Discussion

There is a variable amount of information that can be output to the standard output console (e.g. NSLog). These values define the different levels. Note that a given log level also includes all messages of a lower level, e.g. 'GtarControllerLogLevelInfo' will also output all 'GtarControllerLogLevelWarn' and 'GtarControllerLogLevelError' messages.

GtarControllerThread

The threads that 'GtarControllerObserver' selectors can run on. 

enum GtarControllerThread { 
    GtarControllerThreadMain = 0, 
    GtarControllerThreadMidi 
};
Constants
GtarControllerThreadMain

Notify the observers on the main thread when the current run loop completes.

GtarControllerThreadMidi

Notify the observers immediately (asynchronously) from the MIDI background thread. Note that many iOS Frameworks (e.g. UIKit) will not work properly when running on anything other than the main thread.

Discussion

The GtarController communicates with the Gtar device using the iOS CoreMIDI framework. The MIDI messages from the device are handled on a background thread, asynchronously to the main run loop. The GtarController can immediately notify observers directly from the MIDI background thread, or it can queue up notifications to run on the main thread when the current run loop (if any) completes. Although notifications from the MIDI thread are going to be delivered sooner, there may be concurrency issues associated with asynchronous delivery unless the receiving observer has been designed to handle such occurances.

GtarControllerEffect

The effects available to the Gtar device. 

enum GtarControllerEffect { 
    GtarControllerEffectNone = 0, 
    GtarControllerEffectFretFollow, 
    GtarControllerEffectNoteActive, 
    GtarControllerEffectFretFollowNoteActive 
};
Constants
GtarControllerEffectNone

No special effects.

GtarControllerEffectFretFollow

When the user presses a string into contact with a fret, the underlying LED is lit. The LED is turned off when the string and fret are unpressed.

GtarControllerEffectNoteActive

When a note is plucked, the underlying LED is lit.

GtarControllerEffectFretFollowNoteActive

Both Fret Follow and Note Active effects are active.

Discussion

There are several special effects available on the Gtar device.
